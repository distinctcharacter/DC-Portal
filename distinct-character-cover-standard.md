# Distinct Character Cover Standard

## Purpose

All Distinct Character protocol companions, foundation resources, and resource-library documents must use a cohesive cover system before publication or portal upload.

## Approved Cover Direction

- White background.
- Dark purple primary title.
- Gold accent rule and asset ID.
- Slate grey metadata and legal text.
- Black document-type label when needed.
- Centered composition.
- No legacy headers or footers on the cover page.
- No crowded cover text.
- No pastel, wellness-influencer, or generic coaching styling.

## Cover Hierarchy

1. Brand eyebrow: `DISTINCT CHARACTER`
2. Series or portal location: examples include `SELF-MASTERY SERIES | PHASE 1`, `COGNITIVE ARCHITECTURE | IOS-1`, or `DISTINCT CHARACTER RESOURCE LIBRARY`
3. Gold horizontal rule
4. Primary resource title
5. Document type: `COMPANION MATERIALS`, `FOUNDATION COMPANION`, `PROTOCOL`, or `REFERENCE GUIDE`
6. Short contents descriptor
7. Permanent asset ID
8. Legal entity line
9. Copyright line when appropriate

## Color Tokens

| Token | Use |
|---|---|
| Dark purple | Main title |
| Gold | Series line, rule, asset ID |
| Slate grey | Brand eyebrow, descriptor, legal line |
| Black | Document type |
| White | Page background |

## Implementation Rule

Use `scripts/apply_distinct_character_cover_system.py` to apply the standard. Do not manually redesign each cover unless the document is a special premium edition requiring a separate approved template.
